Background

Font(실습 동안)
	- 무조건 Noto-Sans로 고정
		* font weight, size, 공간으로 디자인을 잡기위해
	- font에 color는 black으로 고정(이외 다른 색깔 금지)

font
	- 한글은 자간을 붙여쓰는 것이 가독성에 좋다.(-25로 고정)
	- 줄간격(line-height): (auto)로 지정하는데, 포토샵에서는 1.2정도이다.

Guideline
	- Guideline 사이의 간격은 30px로 고정

Grid System
	- 웹 페이지를 12개의 column으로 분리하여 text/img를 구성하는 기본적인 시스템
		* 일반적으로 Grid System을 활용하여 레이아웃을 구성하는 수치 : 12 / 6+6 / 4+4+4 / 3+3+3+3

- 화면은 Monitor Size와 동일하게 맞춘다( 1900px ) 
	- 1200px 공간을 따로 설정한다.(수치는 변동가능성 있음)
		* 해당 공간에 image가 삽입된다
		* 각 img들은 1200px을 벗어나선 안된다.

- 라인 맞추기 
	- Grid System을 기반으로 라인을 무조건 맞춰야한다.


이미지
	- 반응형을 위해 사이트에 들어가는 이미지는 모바일 뷰포트의 2배 크기에 해당하는 사이즈가 필요하다(640px, 720px)
	- 이미지가 무조건 크다고 좋은 건 아니다.

- 기타
 	- 쇼핑몰/패션 사이트의 경우 섹션 내 패딩이 늘어나고 글자크기가 작아지는 경우가 있다.
	- 관공서/대학교 사이트의 경우 섹션 내 패딩이 좁아진다.(많은 정보를 전달하기 위함)

텍스트 관련
	1. 강조하고 싶은 텍스트를 위에 쓰고 부가설명은 폰트를 줄여 그 아래에 쓴다
	2. 사람들은 16px 폰트 사이즈에 익숙하기 때문에 기본 폰트 사이즈를 기준으로 사이즈를 조절하는 것이 좋다.
		-> Noto sans korea 의 경우 홀수 사이즈(15px)가 좋다. - 가독성이 좋아짐
	3. 한 섹션 내에서 글자의 정렬은 일관성있게 설정해주는 것이 좋다(left로 정렬했다면 전부 left로, left 썼다가 center 쓰고 이러면 안됨)
	4. 너무 작지도, 크지도 않은 적절한 font-size를 사용하여야 한다.
	5. font-weight, line-height, letter-spacing과 관련된 설정만 작업해주는 것이 좋다
		-> 이외의 것은 설정하지 않는다.
	6. footer에서는 텍스트 줄간격을 붙여주는 것이 좋다.
	7. 사진 옆에 붙는 텍스트의 경우 사진 상단 라인과 딱 맞추게 될 경우 글자가 라인을 벗어난 것처럼 보인다.
	8. 배경화면의 컨텐츠가 전체적으로 가운데 정렬이 되어 있다면 텍스트(슬로건/타이틀)도 가운데로 정렬하는 것이 좋다.
	9. 텍스트의 의미와 역할에 따라 그 크기를 적절하게 지정해줘야 한다.
	10. 한글의 경우 letter-spacing을 의도적으로 줄여주는 것이 가독성에 좋다(보기에도 이쁘다)
		-> letter-spacing으로 많이 띄우면 별로 안예쁘다
	11. 한글과 영어를 섞어서 쓸 때에는 영어에 <span>을 써서 처리해준다.
		-> <span> 처리된 부분은 letter-spacing 처리를 해준다.
	12. anti-aliasing은 항상 sharp로 처리한다.
	13. 아이콘 사이즈는 폰트 사이즈보다 작게 설정 해야한다.
	14. 텍스트가 없는 동영상의 경우 텍스트를 넣어도 되지만, 텍스트가 있을 경우 텍스트는 반드시 제외한다.
	15. 텍스트를 작게하는 것은 좋지만 가독성은 확보되어야 한다.
	16. line-height값은 font-size 이상 띄워서는 안된다.
		-> font-size 이상 설정 시, 한 줄 한 줄이 개별 문장으로 인식된다
	17. 텍스트 사이즈가 작은데 font-weight가 굵으면 가독성이 떨어진다

텍스트 섀도우
	1. 텍스트 섀도우는 들어간듯, 안들어간듯 최대한 희미하게 사용한다

공간
	1. 컨텐츠를 설명하는 타이틀이라면 서로 간의 공간은 좁혀주는 것이 좋다.
	2. 섹션은 한 바닥에 다 보이게 하여 컨텐츠를 한 눈에 보여주는 것이 중요하다.
	3. 서로 다른 섹션 간에 부여하는 마진에 동일한 값을 줘야한다.
	4. 같은 기능/의미가 있을 경우 공간을 좁혀주는 것이 좋다.
	5. height:100vh -> 포토샵 height:970px
	6. 버튼의 경우 버튼 내부의 좌우 패딩을 여유롭게 주어 버튼과 텍스트의 간격을 많이 주는 것이 좋다
	7. Line & Text / Text & Image 사이는 붙이지 않는다.
	8. 섹션에 들어가는 콘텐츠가 한 쪽으로 쏠릴 경우  가이드라인을 조금 넓게 잡아 공평하게 배분한다.(width가 1200px을 넘어갈 수 있다.)
	9. 적정한 margin 값: 100 -10 - 30
	10. 상/좌 공간이 일정해야한다. (좌우 패딩과 상하 패딩이 일정하지 않으면 Unbalance하게 보여 좋지 않다)

색상
	1. 검은색의 경우 실제 웹 브라우저에서는 번져서 가독성이 떨어지는 경우가 생긴다.
	2. 흰색 배경 -> 검은색이 아닌 조금 어두운 회색 
	     어두운 배경 -> 흰색이 아닌 밝은 회색
		-> 흰색 배경:검은색 선 / 검은색 배경:흰색 선으로 설정할 경우 실제 웹 브라우저 화면에서 볼 때 2px로 보이기 때문에 피하는 것이 좋다.
	3. 회색 배경에 검은색 글자/아이콘은 별로 좋지 않다.
	4. 흰색 배경에 검은색 선을 쓸 경우 계단 현상이 생겨 보기에 좋지 않다.
